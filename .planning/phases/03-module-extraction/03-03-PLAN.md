**inky_photo_frame.py (the shim):**

REPLACE the entire 1216-line monolith `inky_photo_frame.py` at the repo root with a 3-line backward-compatible shim:

```python
#!/usr/bin/env python3
"""Inky Photo Frame - backward-compatible entry point for systemd service."""
from inky_photo_frame.app import InkyPhotoFrame
InkyPhotoFrame().run()
```

**Why this works (from research):**
- When systemd runs `python /home/pi/inky-photo-frame/inky_photo_frame.py`, Python executes this file directly
- The `from inky_photo_frame.app import InkyPhotoFrame` triggers the import system
- Python finds the `inky_photo_frame/` package directory (which takes precedence over the `.py` file for imports)
- The shim never imports itself -- it imports the package
- The systemd `ExecStart` path is unchanged: `python inky_photo_frame.py`

**Critical rules:**
- The shim file MUST keep the `#!/usr/bin/env python3` shebang (for direct execution)
- The shim is exactly 4 lines (shebang, docstring, import, call) -- this counts as "3-line shim" (shebang is not a code line)
- Do NOT add `setup_logging()` to the shim -- it is called inside `InkyPhotoFrame.__init__()` in app.py
- Do NOT add `if __name__ == '__main__'` guard to the shim -- it always runs when executed directly
- Do NOT add `sys.path` manipulation -- the import system handles everything naturally
